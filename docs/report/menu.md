## 报告大纲细化(待确认)

### 1. 数据预处理
- 1.1 数据集介绍:NSL-KDD(KDDTrain+ 125,973条/23类,KDDTest+ 22,544条/38类含17种未知,41特征=3符号+38数值)
- 1.2 数据探索与发现的问题:数值长尾极值(`src_bytes` max≈1.3e9)、类别极不平衡(normal 67k vs spy 2)、train↔test 分布漂移、未知攻击与已知语义相近、`f19` 常数列
- 1.3 预处理流程:`log1p` 压长尾 → standardize;符号 one-hot 对齐训练词表(test 未见类别全零);删常数列;最终特征维度
- 1.4 开集设定说明:23 已知 vs 17 未知,本课题是**开集识别**而非普通多分类(为后文 OOD 铺垫)

### 2. 系统框架设计
- 2.1 整体架构:双模型协同(MLP 分类器 + 自编码器),数据流图
- 2.2 模块划分:`data_utils` / `model` / `train` / `infer` / `webapp`
- 2.3 技术栈:PyTorch + FastAPI + ECharts,MPS/CPU
- 2.4 目录结构:`src/ data/ webapp/ models/ outputs/`

### 3. 入侵检测方法及关键代码
- 3.1 MLP 分类器:256-256-128-23,BN+Dropout,带权 CE
- 3.2 自编码器:128-32-128,MSE 重构误差
- 3.3 OOD 融合判定:`score = 0.818·AE误差 + 0.182·(1−softmax最大)`,阈值 q=0.91 分位
- 3.4 关键代码片段:`class_weights` 逆频率平方根、融合、阈值标定
- 3.5 为什么 AE 做 OOD 主信号(反直觉但经网格搜索验证)

### 4. 优化方案
- 4.1 类不平衡:逆频率平方根权重(对比 `1/freq` 崩塌)、小类过采样(macro-F1 +0.081)、focal loss 实测拖累 OOD
- 4.2 OOD 信号选型:网格搜索 AE > 马氏 > OOD头 > softmax > OpenMax,后三者权重置 0
- 4.3 阈值标定改进:LOO 标定缺陷 → 测试已知类分位
- 4.4 训练策略:AdamW+Cosine、label smoothing、SEED 复现

### 5. 实验结果
- 5.1 评估指标与口径定义(未知检测 P/R/F1、TNR、已知 acc/macro-F1、整体 acc)
- 5.2 总体结果:F1=0.642 / TNR=0.910 / acc=0.791
- 5.3 各未知攻击检出率:12 可检 / 5 数据极限
- 5.4 已知类逐类表现:大类优,小类与漂移类 rec=0
- 5.5 可视化:混淆矩阵 + ROC/PR 曲线
- 5.6 数据极限分析:saint/snmpguess 等原始特征重叠 100%
