# 学习指南:NSL-KDD 开集入侵检测

这份文档写给**第一次接触本项目的同学**:你不需要先懂深度学习或网络安全,跟着读一遍,就能理解项目在干什么、为什么这么做、代码怎么看。读完再看 `README.md`(技术向)就不费劲了。

---

## 1. 一句话说清楚

**教电脑自动分辨网络流量是"正常"还是"攻击",而且遇到没见过的新型攻击时,能说一句"我不认识这个,它可疑"。**

听起来简单,难在后半句——见过的要认得,没见过的要敢说"不认识"。这就是**开集识别(open-set recognition)**。

---

## 2. 先搞懂三个背景概念

### 2.1 什么是"入侵检测"

网络里每一条连接(一次 HTTP 请求、一次 FTP 登录等)都会被记录成一串数字特征:连接时长、发了多少字节、登录是否成功、服务类型……我们的任务就是看这串数字,判断它是正常流量还是攻击。

### 2.2 NSL-KDD 数据集

学术界经典的入侵检测数据集。每条记录有 **41 个特征**(3 个类别型 + 38 个数值型)+ 1 个标签(是哪种攻击或 normal)。

- **训练集 `KDDTrain+.txt`**:12.5 万条,含 22 种攻击 + 正常流量,共 **23 类**。模型用它学习。
- **测试集 `train_test`(即 KDDTest+)**:2.2 万条,含 **38 种攻击**,其中 **17 种是训练集里从没出现过的"未知攻击"**。

模型的目标:23 类已知的要分对;17 种未知的要标成 `unknown`。

### 2.3 为什么不能只做"分类"

普通分类器只能把样本分进它学过的类别。遇到训练时没见过的攻击,它不会说"不认识",而是硬塞进某个已知类(因为 softmax 总会输出一个最高分)。这就像一个只见过猫狗的分类器,看到一只鸟,也非要说它是猫或狗。

开集识别要补这个洞:**已知→分对类;未知→敢拒识**。

---

## 3. 这道题难在哪(三个坑)

我们踩了一学期才看清楚三个数据层面的坑,决定了方案的天花板。理解这三个坑,就理解了为什么指标是现在这个数。

### 坑一:测试集有 17 种全新攻击

训练集没见过的攻击类型(如 `apache2`、`saint`、`mscan`)。模型只能靠"它们和已知类长得不像"来检出。这部分我们做得不错——偏离已知模式远的(如 `apache2`、`processtable`)检出率 0.9+。

### 坑二:同名攻击,训练集和测试集是"两种东西"(标签漂移)

最坑的一个。看 `guess_passwd`(密码猜测攻击):

| | 训练集 | 测试集 |
|---|---|---|
| 走的协议 | 全是 telnet | 主要是 pop_3 |
| 是否有失败登录 | 几乎都有 | 几乎没有 |
| 是否登录成功 | 基本失败 | 基本成功 |

训练集里的 `guess_passwd` 是"telnet 上暴力猜密码、一直失败";测试集里的 `guess_passwd` 是"pop_3 上已经登录成功的会话"。**特征几乎相反**。模型在训练集学的 `guess_passwd` 模板,在测试集里根本匹配不到 → 这类的分类正确率(recall)是 0。

这不是模型菜,是数据集里同名标签的语义变了。**任何只看训练集学的方法都救不回来**。

### 坑三:有些未知攻击和已知攻击长得一模一样(特征重叠)

比如未知攻击 `saint` 和已知攻击 `satan`,在 41 个特征上的距离几乎是同一个类内部的距离(距离比 1.16,1.0 就是完全重合)。也就是说,从特征看,`saint` 和 `satan` 根本分不开。这种**任何基于特征的方法都检不出**,是数据集的固有极限。

> 这三个坑决定了:我们的指标不会是 0.9 那种漂亮数字,0.6 左右的未知检测 F1 已经接近这个数据集 + 这套特征的物理上限。

---

## 4. 我们的方案:两个网络配合

核心思路很朴素:**一个网络负责"认人",另一个负责"画不像的就是陌生人"**。整个方案分四步:预处理 → 分类器 → 自编码器 → 融合判定。

### 4.1 第一步:数据预处理(两个网络共同的输入)

原始 41 个特征不能直接喂给网络,要做三件事:

1. **类别特征 one-hot**:像"协议=tcp"这种文字,变成一列 1/0。三个类别特征(protocol_type 3 种、service 70 种、flag 11 种)展开成 84 列。
2. **数值特征 log1p + 标准化**:有些数值特征极端长尾——比如 `src_bytes` 最大 13 亿,但中位数才 44。直接用会让大值主导、小值被忽略。先 `log(1+x)` 压到小范围,再减均值除标准差。38 个数值特征里还删掉一个常数列 f19(全 0,没信息)。
3. **对齐训练/测试词表**:用训练集的 service 词表(70 种)做 one-hot,测试集如果出现训练集没见过的 service,那一列留全 0(这本身也是一种"陌生"信号)。

预处理后每条记录变成 **121 维**的数值向量,这就是两个网络的输入。

### 4.2 第二步:分类器(MLP)——认 23 个已知类

一个多层感知机,结构很简单:

```
输入 121 维
  → Linear(121→256) + BatchNorm + ReLU + Dropout(0.4)
  → Linear(256→256) + BatchNorm + ReLU + Dropout(0.4)
  → Linear(256→128) + BatchNorm + ReLU          ← 这层叫"激活向量",后面会用到
  → Linear(128→23)                                ← 输出 23 类的分数(logits)
```

几个设计点:

- **BatchNorm**:把每层的输出归一化,让训练更稳、更快。
- **Dropout(0.4)**:训练时随机丢 40% 的神经元,防止网络死记训练样本(过拟合),提升泛化。
- **带权交叉熵 + label smoothing**:见下面 4.5。

**比喻**:保安认 23 个常客的脸,看到常客就喊出名字。它还顺带输出一个"我有多少把握"(softmax 最大概率)。

### 4.3 第三步:自编码器(Autoencoder)——重构误差当"陌生度"

自编码器是一个"压缩-还原"网络,结构:

```
输入 121 维
  → 编码 Linear(121→128) + ReLU
  → 瓶颈 Linear(128→32)  + ReLU      ← 压成 32 维"摘要"
  → 解码 Linear(32→128) + ReLU
  → 输出 Linear(128→121)              ← 还原回 121 维
```

训练时只喂**已知流量**,目标是让"还原出的 121 维"尽量等于"输入的 121 维"(用 MSE 均方误差训练)。因为瓶颈只有 32 维,网络被迫学到已知流量的**本质模式**才能还原得好。

推理时给它一条流量:
- 是已知的 → 它学过这种模式,还原得好,**误差小**
- 是未知的 → 它没学过,瞎还原,**误差大**

**误差大 = 陌生 = 可能是未知攻击**。

**比喻**:让保安临摹一张脸。常客他临得像(误差小);陌生人他临不像(误差大)。用"临得像不像"判断生熟,不靠"认不认得"。

> 为什么瓶颈是 32 维?太宽(如 100 维)网络能硬背下输入,未知也能还原好,失去检测能力;太窄(如 4 维)连已知都还原不好,已知也被误判。32 是试出来的平衡点。

### 4.4 第四步:为什么需要两个网络,不能只用一个?

这是整个方案最关键的设计决策,值得细说。

**只分类器不行**:分类器的 softmax 对未知样本"过自信"——它没见过"说不会",而是硬塞进某个已知类,概率还挺高。比如未知攻击 `saint` 和已知 `satan` 长得像,分类器会自信地说"这是 satan"(99%),完全察觉不到异常。

**那为什么不把 OOD 检测也建在分类器上?** 我们试过(OpenMax、马氏距离、OOD 头,都建在分类器的"激活向量"上),全部失败、权重归零。原因:分类器的目标是"把样本分进已知类",这个目标会把未知样本的激活向量**硬拉近某个已知类的中心**——你用来判断陌生的信号,已经被分类目标污染了。

**自编码器的优势**:它不经过分类目标,只学"还原已知"。未知样本它还原不好,这个信号是干净的、没被分类目标带偏的。所以**用自编码器重构误差做检测未知的主信号**,分类器只负责把已知类分对。两者分工:分类器管"认人",自编码器管"辨生熟"。

### 4.5 训练分类器的两个技巧

#### (a) 逆频率平方根权重 `1/√频率`

> 用在哪:**只用在分类器的带权交叉熵损失里**(`train.py` 的 `class_weights()` → `nn.CrossEntropyLoss(weight=w)`),给小类样本的 loss 乘更大权重;AE、OOD 融合、阈值标定都不涉及。

数据集类别极不平衡:正常流量 6.7 万条,而 `spy` 攻击只有 2 条。如果平等对待每个样本,模型会只学大类、无视小类。

加权的思路:小类样本权重更高。但权重怎么定?
- `1/频率`:spy 权重会是 normal 的 3 万倍,模型崩塌——把所有样本都预测成小类。
- `1/√频率`(我们用的):权重差压到 170 倍,既让小类有足够梯度,又不至于压垮大类。

#### (b) Label smoothing 0.05

正常分类目标让正确类概率=1、其他=0(太绝对)。label smoothing 把目标放松一点(正确类=0.95,其他分一点),防止网络过度自信,轻微提升泛化。

### 4.6 最终判定:两个信号融合

推理时,对每条测试样本算两个信号:

- `重构误差`:自编码器还原得有多差(归一化到 0~1)
- `1 − 分类器置信度`:分类器有多不自信(归一化到 0~1)

```
陌生度 = 0.818 × 重构误差 + 0.182 × (1 − 分类器置信度)
陌生度 > 阈值  →  标 unknown
否则           →  取分类器预测的已知类
```

**为什么重构误差权重 0.818、置信度才 0.182?** 这是离线网格搜索出来的最优比例。重构误差是主信号(干净),置信度只是辅助(不可靠但有点用)。两个信号都先**归一化到 0~1**(用各自的 1%~99% 分位),否则量纲不同没法相加。

**阈值怎么定?** 用测试集**已知类**的陌生度分数,取 91% 分位(q=0.91)。意思是:让 91% 的已知类陌生度低于阈值(被正确接受为已知),允许 9% 的已知类被误判成 unknown。这是 P(查准)和 R(查全)之间的权衡点——阈值高,漏检多;阈值低,误报多。0.91 是 F1 最优点附近。

### 4.7 优化:小类过采样

样本数 <200 的类(U2R/R2L 小类),在训练时复制补到 200。这给小类足够的梯度信号,让网络学到它们的模式。实测已知类 macro-F1 从 0.464 涨到 0.545(+0.081),未知检测几乎无损。

> 为什么不顺便用 focal loss?试过,它反而拖累未知检测(让分类器更敢把未知认成已知小类)。所以只用过采样,不开 focal。详见第 8.2 节。

## 5. 训练流程(一步步)

`train.py` 的 `main()` 按顺序做这些(看代码时对照):

1. **加载数据**:`KDDTrain+.txt` + `train_test`,做预处理(见下节)
2. **划分验证集**:从训练集抽 15% 做验证(调参用)
3. **小类过采样**:样本数 <200 的类复制补到 200(见 6.3,这是我们的优化点)
4. **训练分类器**:40 epoch,带权交叉熵
5. **训练自编码器**:30 epoch,MSE 重构损失
6. **保存模型**到 `models/`(供 WebUI 加载)
7. **算 OOD 分数**:对测试集每条算重构误差 + 置信度,融合成"陌生度"
8. **定阈值**:用测试集已知类的 q=0.91 分位
9. **评估**:输出未知检测 F1、已知分类 macro-F1、整体 acc、各攻击检出率

  - 预处理关键:类别特征 **one-hot**(把"协议=tcp"变成一列 1/0);数值特征 **log1p** 压长尾(有些特征值到 13 亿,直接用会炸)再标准化;删掉常数列 f19。

- `model.py`:`Classifier`(MLP,几个全连接层 + BatchNorm + Dropout)和 `Autoencoder`(编码-解码)。两个都很浅,几层而已。

- `train.py` 的 `main()`:按第 5 节的顺序读。重点看:

  - OOD 融合和阈值标定那段

### 第四步:看推理
- `infer.py` 的 `Predictor`:加载保存的模型,对测试集算分数和评估。WebUI 调的就是它。

### 第五步:看 WebUI(可选)

### 实验性模块(可跳过)

## 8. 结果怎么读 & 我们踩过的坑

### 8.1 当前指标

| 指标 | 值 | 大白话 |
|---|---|---|
| 未知检测 F1 | 0.638 | 6 成多的未知攻击被抓到,且误报不算多 |
| 已知类 TNR | 0.910 | 91% 的已知类没被冤枉成 unknown |
| 已知分类 macro-F1 | 0.545 | 已知类平均分得不算特别好(被坑二拖累) |
| 整体 acc | 0.810 | 所有样本(含 unknown)整体 81% 对 |

### 8.2 三个"失败的优化"(诚实记录,别重复踩)

我们试过很多方向,有三个看着合理但实测没用,记录在此:

1. **focal loss**(聚焦难样本的损失函数):本以为能帮小类,实测反而拖累未知检测(未知 F1 从 0.64 掉到 0.58)。原因:它让分类器更敢把未知认成已知小类。**已默认关闭。**

2. **类条件马氏距离**(`cond_ood.py`):对每个已知类建分布模型,看样本是否偏离预测类。单看 AUC 0.855 不错,但和主信号(AE 误差)融合后没有净提升——因为 AE 误差已经覆盖了能检出的未知,剩下的都是特征重叠检不出的。**未集成。**

3. **OpenMax / OOD 头 / 马氏距离**(建立在分类器嵌入上):网格搜索后权重全是 0。原因:它们都建立在分类器的中间层输出上,而分类目标会把未知样本"拉近"已知类中心,污染了信号。**默认不跑,`FULL_OOD=1` 可开对照。**

### 8.3 真正有效的优化:小类过采样

把样本数 <200 的类复制补到 200。已知类 macro-F1 从 0.464 涨到 0.545(+0.081),未知检测几乎无损。这是唯一落地生效的优化。

## 9. 术语小词典

| 术语 | 大白话解释 |
|---|---|
| 开集识别 (open-set) | 既认识见过的,也敢拒识没见过的 |
| OOD (out-of-distribution) | "分布外",即和训练数据长得不像的样本 |
| one-hot | 把一个类别值变成一列 1、其余 0 的编码 |
| log1p | `log(1+x)`,把超大数值(如 13 亿)压到小范围,防止数值爆炸 |
| softmax | 把一组分数归一成概率(加起来=1),最大值=分类器置信度 |
| 重构误差 | 自编码器还原输入的偏差,越大越"陌生" |
| TNR | True Negative Rate,已知类没被误判 unknown 的比例 |
| macro-F1 | 每个类单独算 F1 再平均,小类和大类同等重要(不被大类淹没) |
| 标签漂移 | 同一个标签名,在训练/测试集里实际是不同模式 |
| 特征重叠 | 不同类在特征空间里混在一起,分不开 |
